    ARM 板端部署¶

    在 Jetson AGX Orin 上不依赖 Docker，用 conda 环境 env_vla（Python 3.12）运行天工 LeRobot 的推理/训练部署。

    适用环境¶

    • 适用目录：ubt_IL/scripts/deploy/tienkung_pro/arm_64/
    • 目标设备：Jetson AGX Orin（JetPack 6 / L4T R36.4.0 / Ubuntu 22.04 / glibc 2.35 / CUDA 12.6）
    • 设计原则：不 sudo、不建软链、不改原脚本

    运行架构（双 Python 栈）¶

    本部署包的核心是两个 Python 栈分工：env_vla (3.12) 跑 LeRobot 推理/训练，系统 python3.10 跑只能用 cp310 的硬件相关服务。两者通过 ZMQ 解耦，互不污染。

    env_vla（conda, Python 3.12）:  LeRobot + tienkung 插件 + torch
   │  lerobot-rollout 推理 / lerobot-train 训练 / image_client 通路验证
   │
   ├── 5558 ◄── ImageServer（系统 py3.10 + pyorbbecsdk）──► 相机
   │
   └── 5559 ──► Bridge2（系统 py3.10 + ROS2）──ROS2 DDS──► 天工硬件
       5560 ◄──┘
    ZMQ 端口 方向 用途 由谁启动
    5558 ImageServer → env_vla 相机 JPEG 帧 image_server_host.sh（手动，常驻）
    5559 env_vla → Bridge2 动作指令（action） 插件在 rollout 时自动拉起 Bridge2
    5560 Bridge2 → env_vla 机器人状态（status） 同上

    两个栈的归属：

    • env_vla (3.12)：rollout_host.sh、train_host.sh、image_client_host.sh。运行前需 conda activate env_vla。
    • 系统 python3 (/usr/bin/python3 3.10.12)：image_server_host.sh、robot_ready.sh，以及 Bridge2（由插件用 /usr/bin/python3 自动启动）。

    文件清单¶

    文件 作用 运行栈
    setup_env.sh 一键构建 conda 环境（Python 3.12 + wheel + LeRobot + 插件） 构建器
    rollout_host.sh 策略推理部署 env_vla (3.12)
    train_host.sh ACT 模型训练（可选） env_vla (3.12)
    image_server_host.sh 相机图像服务：相机 → ZMQ 5558（JPEG） 系统 python3.10
    image_client_host.sh 相机通路验证客户端 env_vla (3.12)
    robot_ready.sh 机器人复位到预设位置 系统 python3.10
    torch-*.whl 本地编译的 PyTorch/torchvision wheel（Jetson 专属） —

    使用流程¶

    前置条件¶

    • 项目代码位于 /home/nvidia/vla/TienKung-IL-LAB/
    • 已安装 conda（miniconda / miniforge）
    • 宿主机已有 ROS2 Humble（/opt/ros/humble）
    • 系统 python3.10 已装 pyorbbecsdk2 / cv2 / zmq / numpy
    • 真机部署：把 ubt_IL/docker/fastdds_no_shm.xml 中的 IP 改为本机实际 IP

    构建环境（首次）¶

    cd /home/nvidia/vla/TienKung-IL-LAB/ubt_IL/scripts/deploy/tienkung_pro/arm_64
bash setup_env.sh          # 创建 conda env_vla，装 wheel + LeRobot + 插件并自检
    变量 默认值 说明
    CONDA_BASE conda info --base 或 ~/miniconda3 conda 安装根目录
    ENV_NAME env_vla conda 环境名
    PIP_MIRROR 清华 TUNA pip 镜像

    标准部署顺序（每次推理）¶

    # ① 相机服务（系统 python3.10，独立终端 / 后台常驻）
bash image_server_host.sh

# ②（可选）机器人复位到预设位置
bash robot_ready.sh

# ③ 策略推理（env_vla 3.12，新终端）
conda activate env_vla
POLICY_PATH=/home/nvidia/vla/TienKung-IL-LAB/ubt_IL/model/Pick_up_tiangong_all_act/checkpoints/last/pretrained_model \
  DURATION=60 bash rollout_host.sh

# 部署 13-DOF 模型
JOINT_CONFIG=tienkung_13 \
POLICY_PATH=/home/nvidia/vla/TienKung-IL-LAB/ubt_IL/model/sim_pick_place_right13_act/checkpoints/last/pretrained_model \
  DURATION=60 bash rollout_host.sh

DISPLAY_CAM=false bash rollout_host.sh        # SSH 无 X 时关相机显示

    Rollout 参数¶

    变量 默认值 说明
    POLICY_PATH $PROJECT_ROOT/model/.../pretrained_model ACT checkpoint
    JOINT_CONFIG tienkung_26 关节 DOF 配置（须与训练 DOF 一致）
    STRATEGY base 推理策略
    TASK sim_pick_place 任务描述
    ZMQ_HOST 127.0.0.1 image_server 地址
    DURATION 60 运行时长（秒）
    FPS 30 控制环频率
    DISPLAY_CAM true 相机显示（SSH 无 X 设 false）

    验证相机通路¶

    bash image_client_host.sh --count 60               # 收 60 帧后退出并给结论
bash image_client_host.sh --show                   # 弹窗预览（需 X）
bash image_client_host.sh --address 192.168.41.2   # 连真机上的 image_server

    输出 => 相机通路 OK ✓ 即正常。

    模型训练（可选）¶

    conda activate env_vla
bash train_host.sh
STEPS=100000 BATCH_SIZE=8 bash train_host.sh
    变量 默认值 说明
    CONFIG_PATH train_config_tiangong_all.json 训练配置
    DATASET_ROOT $PROJECT_ROOT/dataset LeRobotDataset root
    DATASET_REPO_ID Pick_up_the_apple_all 与实际数据集目录名一致
    OUTPUT_DIR $PROJECT_ROOT/model/Pick_up_tiangong_all_act 输出目录
    STEPS 500000 训练步数
    BATCH_SIZE 8 批大小
    SEED 10000 随机种子
    DEVICE cuda 训练设备

    数据集回放（可选）¶

    /usr/bin/python3 $PROJECT_ROOT/scripts/deploy/replay.py \
    --dataset $PROJECT_ROOT/dataset/Pick_up_the_apple_all --episode 0 --rate 30

    关键约束¶

    Python 3.12

    LeRobot 0.5.2 + tienkung 插件原生运行，免源码补丁；勿降级 3.10。

    本地 wheel

    torch/torchvision 用本目录的 cp312 Jetson 专属 wheel（链接本机 glibc 2.35）。 勿用 PyPI 的 aarch64 wheel（CPU-only、无 CUDA），也勿用需 GLIBC_2.38 的预编译 wheel（本机 2.35 会崩）。

    numpy 必须 <2

    本地 torch 按 numpy 1.x 编译，装 2.x 会触发 "compiled using NumPy 1.x" 崩溃。 setup_env.sh 已强制 numpy==1.26.4，勿手动升级。

    运行栈别混

    image_server_host.sh、robot_ready.sh 走系统 python3.10，不需要 conda activate env_vla； rollout_host.sh、train_host.sh、image_client_host.sh 走 env_vla (3.12)，需先 activate。

    排错¶

    • 相机显示：DISPLAY_CAM=true 与 image_client --show 需 X 环境；SSH 无 DISPLAY 时设 false 或用 ssh -X/-Y。
    • conda activate 失效：非交互 shell 中先 source $CONDA_BASE/etc/profile.d/conda.sh 再 conda activate env_vla（脚本内已处理）。
    • 磁盘：env_vla 含 torch 约 1.5G，eMMC 紧张；pip 已全程 --no-cache-dir。

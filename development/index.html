    开发指南¶

    项目结构¶

    项目按功能 × 机型组织，ubt_sim/ 负责仿真，ubt_IL/ 负责训练和部署：

    ubt_IL/scripts/
├── convert/              # 数据转换
│   ├── common/           # 共享转换器
│   ├── tienkung_pro/     # 天工配置 + convert.sh
│   └── walker_s2/        # Walker 配置 + convert.sh
├── train/                # 模型训练
│   ├── tienkung_pro/     # 天工训练配置 + train.sh
│   └── walker_s2/        # Walker 训练配置 + train.sh
└── deploy/               # 模型部署
    ├── tienkung_pro/     # 天工 rollout/replay/reset/camera
    │   └── arm_64/       # Jetson ARM 板部署包
    └── walker_s2/        # Walker rollout/camera

    添加新场景¶

    1. 放置 3D 文件到 assets/scenes/<name>/
    2. 创建 config/<task>.yaml（含 scene、robot、cameras、simulation 字段）
    3. 创建 source/ubt_sim/task/<task>/（继承 ManagerBasedRLDigitalTwinEnvCfg）
    4. 在 source/ubt_sim/__init__.py 注册任务

    添加新机器人¶

    扩展新机器人需在仿真侧、场景侧、遥操作侧分别新建模块，并接入启动链路。

    仿真侧（Isaac Sim，Py 3.11）¶

    source/ubt_sim/devices/<name>/
├── __init__.py
├── config.py              # <NAME>_CFG / 关节组常量 / <NAME>_USD_PATH
├── controller.py          # <Name>Controller(DeviceBase)
└── action_process.py      # 动作处理
    • 在 source/ubt_sim/devices/__init__.py 导出 controller 与 config 常量
    • 创建任务包 source/ubt_sim/task/<task>/：
    • __init__.py：gym.register("UBTSim-<Robot>-<Scene>-v0", ...)
    • <task>_env_cfg.py：继承 ManagerBasedRLDigitalTwinEnvCfg，实现 use_teleop_device() / preprocess_device_action()
    • 在 source/ubt_sim/__init__.py 加 from .task.<task> import *

    场景侧¶

    • 机器人模型：assets/robots/<name>/（USD + URDF + textures）
    • 场景资产：assets/scenes/<scene>/（USD）
    • 任务/场景配置：config/<name>/<task>.yaml

    遥操作侧（ROS2，Py 3.10）¶

    teleoperation/
├── bridges/<name>/
│   ├── <name>_ros2_zmq_bridge.py
│   └── <name>_bridge_config.yaml
└── control/<name>/
    ├── reset.py
    ├── <pick_place>_save_data.py
    ├── save_data.sh
    └── constants.py
    • 自定义 ROS2 消息（按需）：源码放 teleoperation/msgs/，容器内构建

    启动链路¶

    • scripts/sim_runner.py：_detect_robot() 识别新任务名；main() 按机器人加 controller 实例化 / physics device 分支
    • scripts/start_sim.sh：任务名 → ROBOT 检测、bridge 启动分支

    部署侧（LeRobot，ubt_IL）¶

    ubt_IL/<robot_slug>/
├── lerobot_robot_<slug>/
│   ├── pyproject.toml
│   └── lerobot_robot_<slug>/
│       ├── __init__.py         # @register_subclass 注册
│       ├── <slug>.py           # Robot 类
│       ├── config_<slug>.py    # LeRobot 配置注册
│       ├── constants.py        # 关节/DOK 枚举
│       ├── hand_utils.py       # 末端执行器控制
│       └── camera/             # 相机集成
├── ros2_<slug>_bridge.py       # Bridge2
├── init_<slug>.py              # 机器人初始化
└── CLAUDE.md                   # 技术文档

    添加新控制任务¶

    1. 继承 DeviceBase（source/ubt_sim/devices/device_base.py），实现 reset() / advance() / add_callback()
    2. 在对应任务 EnvCfg 中实现 preprocess_device_action() 方法完成动作映射

    自定义 DOF 配置¶

    天工 Pro¶

    在 ubt_IL/tienkung/lerobot_robot_tienkung/lerobot_robot_tienkung/constants.py 中：

    class YourConfig(IntEnum):
    """自定义 DOF 配置"""
    JOINT_1 = 0
    JOINT_2 = 1
    # ...

JOINT_INDEX_ENUMS["your_config"] = YourConfig

    Walker S2¶

    在 ubt_IL/walker/lerobot_robot_walker/lerobot_robot_walker/constants.py 中：

    class YourWalkerConfig(IntEnum):
    """自定义 Walker DOF"""
    # ...

JOINT_INDEX_ENUMS["walker_s2_xxd"] = YourWalkerConfig

    命名规范¶

    • 配置文件命名：<function>_config_<machine>_<variant>.json
    • 例：train_config_walker_s2_real_pi05_19d.json
    • 例：tienkung_pro_26d_1RGB.json
    • 机型目录小写 + 下划线：tienkung_pro/、walker_s2/
    • 脚本命名：<function>.sh / convert.sh / train.sh / rollout.sh
    • 环境变量大写 + 下划线：POLICY_PATH、JOINT_CONFIG、DATASET_ROOT

    关键约束¶

    项目几个关键约束，任何改动都需遵守：

    1. Python 版本隔离：不同 Python 版本之间零交叉导入，仅通过 ZMQ/ROS2 DDS 通信
    2. numpy < 2：cv_bridge 兼容性要求
    3. FastDDS no-SHM：容器内即使 --network=host，共享内存传输仍会导致 ROS2 订阅失败
    4. 插件独立性：天工/Walker 插件与 LeRobot 源码相互独立，通过注册机制集成
    5. ROS2 Domain ID：仿真 146，真机 0

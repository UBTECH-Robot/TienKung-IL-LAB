# realsense_wrist_camera

Intel RealSense D405 wrist camera ROS2 publisher for Walker S2.

## Quick Start

```bash
# 1. Install (auto-discovers cameras, generates config, persists PATH)
bash /ubt_IL/realsense_wrist_camera/scripts/install.sh

# 2. Start (background by default)
bash /ubt_IL/realsense_wrist_camera/scripts/start.sh

# 3. Check status
bash /ubt_IL/realsense_wrist_camera/scripts/start.sh --status

# 4. Stop
bash /ubt_IL/realsense_wrist_camera/scripts/start.sh --stop
```

Run in foreground for debugging: `bash start.sh --fg`. All extra arguments
are forwarded to the CLI — e.g. `bash start.sh --fg --discover`.

## Verify

```bash
# Check frames are publishing
ros2 topic echo /sensor/camera/wrist_right/color/raw --once --qos-reliability best_effort
```

## Config

Auto-generated by `install.sh`. To regenerate:

```bash
find-realsense-cameras
```

Manual start with custom config:

```bash
realsense-wrist-camera --config /path/to/cameras.json &
```

Config format (`configs/wrist_cameras.json`):

```json
{
  "cameras": [
    {
      "serial": "261122273724",
      "topic": "/sensor/camera/wrist_right/color/raw",
      "msg_type": "shm_msgs/Image1m",
      "frame_id": "wrist_right",
      "width": 640,
      "height": 480,
      "fps": 15
    },
    {
      "serial": "261022273252",
      "topic": "/sensor/camera/wrist_left/color/raw",
      "msg_type": "shm_msgs/Image1m",
      "frame_id": "wrist_left",
      "width": 640,
      "height": 480,
      "fps": 15
    }
  ]
}
```

## Walker S2 Integration

Publishes `shm_msgs/Image1m` on standard wrist topics. Bridge2 `CameraRelay` picks them up automatically — no extra config.

```bash
# Terminal 1
bash /ubt_IL/realsense_wrist_camera/scripts/start.sh

# Terminal 2 — rollout as usual
POLICY_PATH=/ubt_IL/model/<policy>/checkpoints/last/pretrained_model \
bash /ubt_IL/scripts/deploy/walker_s2/rollout.sh
```

## Setup

### udev rules (host, one-time)

```bash
sudo sh -c 'echo "SUBSYSTEM==\"usb\", ATTR{idVendor}==\"8086\", ATTR{idProduct}==\"0b5b\", MODE=\"0666\"" > /etc/udev/rules.d/99-realsense-d405.rules'
sudo udevadm control --reload-rules
sudo udevadm trigger
```

### Docker

Container must run with `--privileged`. The project `run.sh` already includes this.

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| `No RealSense devices detected` | Check USB + udev rules |
| `ModuleNotFoundError: pyrealsense2` | Re-run `install.sh` |
| `ModuleNotFoundError: rclpy` | `source /opt/ros/humble/setup.bash` |
| `Permission denied` on USB | Docker missing `--privileged` |
